# Louisa's Perfumes - Deployment Checklist

## What Was Set Up

✅ **Supabase Integration**
- Product management with PostgreSQL
- Supabase Authentication (email/password)
- File storage for images
- API endpoints for products and admin

✅ **Serverless API Functions**
- `GET /api/products-db` — Fetch products from Supabase
- `POST /api/auth` — Login/signup with Supabase Auth
- `POST /api/admin-products` — Create products (admin only)
- `POST /api/recommend` — AI recommendations with Gemini

✅ **Admin Panel**
- Email/password based authentication
- Product upload form
- Inventory viewer

✅ **AI Chatbot**
- Powered by Google Gemini API
- Uses only products from Supabase (stays in context)

## Step-by-Step Deployment

### 1. Set Up Supabase Database

**Create the `products` table:**

1. Go to: https://bqnepoxtsfqyuhsvxzby.supabase.co/
2. Click **SQL Editor** → **New Query**
3. Paste and run:

```sql
CREATE TABLE products (
  id BIGINT PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
  name VARCHAR(255) NOT NULL,
  manufacturer VARCHAR(255) NOT NULL,
  ml INTEGER NOT NULL,
  price INTEGER NOT NULL,
  description TEXT NOT NULL,
  notes TEXT[] DEFAULT '{}',
  image_url VARCHAR(500),
  in_stock BOOLEAN DEFAULT true,
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
);

CREATE INDEX idx_products_name ON products(name);
CREATE INDEX idx_products_in_stock ON products(in_stock);

ALTER TABLE products ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Allow public read" ON products
  FOR SELECT USING (true);

CREATE POLICY "Allow authenticated insert" ON products
  FOR INSERT WITH CHECK (auth.role() = 'authenticated');
```

**Create file storage bucket:**
1. Go to **Storage** → **Create new bucket**
2. Name: `product-images`
3. Make it **Public**

### 2. Get Your API Keys

1. Go to: https://bqnepoxtsfqyuhsvxzby.supabase.co/project/settings/api
2. Copy these keys:
   - `SUPABASE_URL` = The URL shown (already: https://bqnepoxtsfqyuhsvxzby.supabase.co)
   - `SUPABASE_ANON_KEY` = Public key
   - `SUPABASE_SERVICE_ROLE_KEY` = Secret key (keep private!)

### 3. Get Gemini API Key

1. Go to: https://ai.google.dev/
2. Click **Get API Key**
3. Create a new API key for Vercel

### 4. Deploy to Vercel

1. Go to your Vercel project dashboard
2. Click **Settings** → **Environment Variables**
3. Add these variables (paste your actual keys):

```
SUPABASE_URL=https://bqnepoxtsfqyuhsvxzby.supabase.co
SUPABASE_ANON_KEY=your_anon_key_here
SUPABASE_SERVICE_ROLE_KEY=your_service_role_key_here
GEMINI_API_KEY=your_gemini_api_key_here
```

4. Click **Save** → **Redeploy**

### 5. Create Your First Admin Account

1. Go to your deployed site → `/admin.html`
2. Click **Sign In Required**
3. For email, use your Supabase email (can be any email)
4. For password, use a strong password
5. Click **Create Account** (first time creates the account)
6. Then login with those credentials

### 6. Upload Your First Product

1. Login to admin panel
2. Fill in:
   - Product Name
   - Manufacturer
   - Volume (ml)
   - Price (NGN)
   - Description
   - Notes (comma-separated, e.g. "vanilla, sweet, floral")
   - Image URL (optional - any public image URL)
   - Check "In Stock" box
3. Click **Save Perfume**

### 7. Test the User Site

1. Go to homepage
2. You should see your product in the collection
3. Try the AI chatbot - ask it for perfume recommendations
4. Try filtering by notes

## Troubleshooting

| Problem | Solution |
|---------|----------|
| "Missing SUPABASE_URL" | Add all 3 env vars to Vercel, then redeploy |
| Admin signup not working | Check Supabase Auth is enabled (Settings → Auth Providers) |
| Products not showing | Verify `products` table exists in Supabase SQL Editor |
| AI chatbot not working | Check GEMINI_API_KEY is correct in Vercel env vars |
| Images not displaying | Use valid HTTPS URLs only |
